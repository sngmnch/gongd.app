# 📱 "사이언스 셔틀" 앱 - 최종 프로젝트 정리

---

## 🎯 **프로젝트 한 줄 요약**

> **"대전의 수·과학 교육 기회를 공평하게 하는 실제 작동하는 모바일 앱"**  
> 학생이 앱을 열어 추천 프로그램을 보고, 클릭 3번으로 셔틀을 예약해서 탈 수 있는 통합 플랫폼

## 📋 **앱의 3가지 핵심 기능**

### 1️⃣ **AI 기반 맞춤형 추천 엔진**
```
학생이 하는 일: 프로필 입력 (학교, 관심분야)
↓
AI가 하는 일: 
  - 지역별 교육 격차 분석
  - 학생의 관심분야 매칭
  - 우선 추천 프로그램 선별
  
결과: 학생을 위한 최상의 프로그램 3-5개 추천
```

예시:
```
학생: 동구 가양중, 로봇 관심
추천:
1. 🌟 [7/13] 대덕고 "로봇 만들기" 워크숍 (88점)
2. ⭐ [7/20] 과학관 "AI 체험" (85점)
3. ⭐ [7/27] ETRI 투어 (80점)
```

---

### 2️⃣ **실시간 셔틀 버스 예약 시스템**
```
학생이 하는 일:
1. 프로그램 선택
2. "예약하기" 클릭
3. 보호자 확인

앱이 하는 일:
- 버스 좌석 자동 배정
- 픽업 시간 설정
- 실시간 위치 추적 URL 생성
- 보호자/학생에게 알림 발송

결과: 토요일 10:00에 학교에서 버스 탑승 🚌
```

---

### 3️⃣ **지역 교육 자원 맵 & 통계**
```
시각화 내용:
- 대전 지도에 각 구별 자원 표시
- 영재고, 과학고, 특성화고 위치 표시
- 구별 교육 자원 점수 비교
- "내 지역과의 격차" 통계

목적: 학생들이 현실을 알고 실천하기
```

## 🏗️ **앱 기본 화면 구성**

```
홈 화면
├─ 👤 프로필 (이름, 학교, 관심분야)
├─ 🌟 나를 위한 추천 프로그램 (상위 3-5개)
│  └─ 각 프로그램: 제목, 날짜, 장소, 셔틀 가능 여부, [자세히] [예약하기]
├─ 📊 이번주 셔틀 버스 목록
└─ 🔔 공지사항

셔틀 예약 화면
├─ 예약 가능한 모든 버스 목록
├─ 각 버스: 프로그램, 시간, 좌석 상황, [예약하기]
└─ 예약 후: 실시간 추적, 보호자 확인 요청

지도 화면
├─ 대전 지도 (5개 구 색상 구분)
├─ 각 구 교육 자원 점수 표시
├─ 클릭 시: 상세 정보 (영재고, 과학고, 동아리, 교원)
└─ "내 지역" 강조 표시

마이페이지
├─ 예약된 프로그램 목록 (예정/진행중/완료)
├─ 참여 기록 및 시간
├─ 후기 작성
└─ 프로필 수정
```

## 💻 **기술 스택 선택**

### 프론트엔드
```
React.js
- 이유: 반응형 UI, 실시간 상태 관리 용이
- 라이브러리: 
  * Mapbox/Kakao Map (지도)
  * Chart.js (통계)
  * React Router (네비게이션)
```

### 백엔드
```
Node.js + Express
- 이유: JavaScript 통일, 빠른 개발, 확장성
- 기능:
  * REST API 제공
  * 추천 알고리즘 실행
  * 셔틀 예약 로직
```

### 데이터베이스
```
PostgreSQL
- 이유: 교육청 공공데이터와 호환, 안정성
- 테이블:
  * students (학생)
  * programs (프로그램)
  * buses (셔틀)
  * bookings (예약)
  * regions (지역)
```

### AI/ML
```
Python + scikit-learn
- 역할:
  * 협력 필터링 (비슷한 학생 찾기)
  * 콘텐츠 기반 추천
  * 자연어처리 (생기부 분석)
```

## 🔄 **사용자 여정 (User Journey)**

### 첫 방문 학생

```
1. 앱 설치
   ↓
2. 회원가입 (이메일, 비밀번호, 학교 선택)
   ↓
3. 프로필 설정
   - 학교: "대전가양중" 선택
   - 관심분야: "수학", "로봇" 체크
   - 주말 이동: "가능" 선택
   ↓
4. AI 추천 결과 보기
   - 지역 격차도 함께 설명
   - "동구는 프로그램이 부족합니다" 정보 제공
   ↓
5. 프로그램 클릭 → 상세 정보 확인
   ↓
6. 셔틀 예약
   - "예약하기" 클릭
   - 보호자 휴대폰 인증
   - 완료!
   ↓
7. 셔틀 대기
   - 실시간 버스 위치 추적
   - 카톡/문자 알림 수신
   ↓
8. 프로그램 참여 후
   - 후기 작성
   - AI가 다음 추천에 반영
```

## 📊 **추천 알고리즘의 마법**

### 점수 계산 공식
```
추천점수 = 
  (관심분야 매칭도 × 40%) +
  (지역 격차 우선도 × 30%) +
  (난이도 적합도 × 15%) +
  (인기도 × 15%)

예시: "로봇 만들기" 프로그램 (동구 학생)
- 관심분야: 0.95 × 40% = 38점
- 지역격차: 0.90 × 30% = 27점
- 난이도: 0.80 × 15% = 12점
- 인기도: 0.85 × 15% = 12.75점
─────────────────────────
최종 점수: 89.75점 ⭐⭐⭐⭐⭐
```

### 왜 동구 학생을 우선할까?
```
동구: 교육 자원 점수 40/100 (낮음)
→ 교육 격차가 크다 = 우선순위 높다

서구: 교육 자원 점수 85/100 (높음)
→ 이미 많은 기회가 있다 = 우선순위 낮다

결과: 같은 프로그램도 동구 학생에게는 더 높은 점수 부여
```

## 💰 **프로젝트 가치**

### 학생 입장
```
기존: 정보 없음 → 기회 놓침
이제: 앱으로 추천 받음 → 셔틀 탈 수 있음
```

### 학교/교육청 입장
```
기존: 교육 격차를 "알고만" 있음
이제: 데이터로 검증하고 셔틀로 해결
```

### 사회 입장
```
기존: 부의 차이 = 교육 기회의 차이
이제: 공공 앱으로 균등한 기회 제공
```

## 🎓 **대회 활용 방안**

### PPT는 "왜"를, 앱은 "어떻게"를 보여줌

```
PPT (발표 심사)
├─ 문제: 교육 격차가 심함
├─ 분석: 데이터로 입증
├─ 솔루션: 3단계 프로세스

앱 (실제 구현)
├─ 프로필 입력해보기
├─ 추천 프로그램 보기
├─ 셔틀 예약해보기
└─ 지도에서 자원 확인하기
```

**심사위원 입장에서:**
```
❌ PPT만: "아이디어는 좋은데 실현 가능한가?"
✅ PPT + 앱: "오! 실제로 작동하는구나!"
```

## 🎯 **최종 성공 기준**

```
✅ 앱이 실제로 작동하는가?
   → 학생이 프로필 입력 → 추천 받음 → 예약 가능

✅ 사용자가 직관적으로 이해하는가?
   → 3번의 클릭으로 셔틀 예약

✅ 데이터 기반인가?
   → 공공데이터 활용, 추천 알고리즘으로 검증

✅ 실제 문제를 해결하는가?
   → 동구·중구 학생도 서구·유성구 프로그램에 접근 가능

✅ 확장 가능한가?
   → 다른 도시에도 적용 가능한 모델
```
