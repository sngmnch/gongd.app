## 📊 **현재 가진 데이터 분석**

### 사용 가능한 데이터
```
1. 고등학교 정확한 위치.csv
   ├─ 학교명, 지역(구), 위도, 경도 ⭐
   ├─ 설립구분(공립/사립), 학교특성(영재고/과학고/특성화고)
   └─ 전화번호, 홈페이지

2. 중학교 정확한 위치.csv
   ├─ 학교명, 지역(구), 위도, 경도 ⭐
   └─ 설립구분

3. 고등학교 동아리 수.csv
   ├─ 지역(구), 동아리 수
   └─ 참여 학생 수

4. 중학교 동아리 수.csv
   ├─ 지역(구), 동아리 수
   └─ 참여 학생 수

5. 교원 현황 데이터
   ├─ 지역(구), 과학/수학 교원 수
   └─ 학교별 특성

6. 영재특목자사진학율.xlsx
   └─ 지역별 진학률 (우선순위 결정)
```

### 3단계: 최적 경로 설계 (핵심!)

#### A. 거리 기반 경로 최적화 (TSP 알고리즘)

**문제 정의:**
```
여러 중학교를 거쳐 고등학교(프로그램 장소)에 도달해야 함
목표: 총 주행 시간 최소화 + 학생 탑승 시간 최소화

제약조건:
1. 총 주행 시간 ≤ 60분 (학생이 버스에 오래 탈 수 없음)
2. 픽업 시간 차이 ≤ 30분 (공평성)
3. 버스 용량 = 40명 (초과 불가)
```

**구체적 예시: "로봇 워크숍" 버스 노선**

```
목적지: 유성구 대덕고 (14:00 프로그램 시작)

픽업할 중학교들:
1. 대전가양중 (동구) - 36.3509, 127.4487
2. 대전가오중 (동구) - 36.3508, 127.4381
3. 대전대문중 (중구) - 36.3320, 127.4073
4. 갈마중 (서구) - 36.3518, 127.3857
5. 대덕중 (유성구) - 36.3874, 127.3814 (= 목적지)

▼ 거리 계산 (위도경도 → 실제 거리)

대전가양중 → 대전가오중: 1.2km (3분)
대전가오중 → 대전대문중: 2.1km (5분)
대전대문중 → 갈마중: 3.8km (10분)
갈마중 → 대덕중(목적지): 4.2km (12분)

총 거리: 11.3km
총 시간: 30분

▼ 최적화된 픽업 시간 설정

13:00 - 대전가양중 출발 (첫 픽업)
13:03 - 대전가오중 도착 (학생 탑승)
13:08 - 대전대문중 도착 (학생 탑승)
13:18 - 갈마중 도착 (학생 탑승)
13:30 - 대덕고 도착 (프로그램 장소)

※ 14:00 프로그램까지 30분 여유 (준비 시간)
```

#### B. Python 코드로 최적 경로 계산

```python
from math import radians, sin, cos, sqrt, atan2
from itertools import permutations

def haversine_distance(lat1, lon1, lat2, lon2):
    """위도경도로 실제 거리(km) 계산"""
    R = 6371  # 지구 반지름
    
    lat1, lon1, lat2, lon2 = map(radians, [lat1, lon1, lat2, lon2])
    dlat = lat2 - lat1
    dlon = lon2 - lon1
    
    a = sin(dlat/2)**2 + cos(lat1) * cos(lat2) * sin(dlon/2)**2
    c = 2 * atan2(sqrt(a), sqrt(1-a))
    
    return R * c  # km 단위

def calculate_route_time(pickup_points, destination):
    """여러 픽업지점을 거쳐 목적지에 도달하는 최단 경로 시간 계산"""
    
    # 모든 가능한 경로 조합 생성
    best_route = None
    best_time = float('inf')
    
    for route in permutations(pickup_points):
        total_distance = 0
        
        # 이전 지점에서 다음 지점까지 거리 계산
        current = route[0]
        for next_point in route[1:] + (destination,):
            distance = haversine_distance(
                current['lat'], current['lon'],
                next_point['lat'], next_point['lon']
            )
            total_distance += distance
            current = next_point
        
        # 시간으로 변환 (시속 30km 기준)
        total_time = (total_distance / 30) * 60  # 분 단위
        
        if total_time < best_time:
            best_time = total_time
            best_route = route
    
    return best_route, best_time

# 실행
pickup_schools = [
    {'name': '대전가양중', 'lat': 36.3509, 'lon': 127.4487},
    {'name': '대전가오중', 'lat': 36.3508, 'lon': 127.4381},
    {'name': '대전대문중', 'lat': 36.3320, 'lon': 127.4073},
    {'name': '갈마중', 'lat': 36.3518, 'lon': 127.3857},
]

destination = {'name': '대덕고', 'lat': 36.3874, 'lon': 127.3814}

best_route, best_time = calculate_route_time(pickup_schools, destination)

print(f"최적 경로: {[s['name'] for s in best_route]}")
print(f"예상 소요 시간: {best_time:.0f}분")
```

## 🤖 **자동 노선 생성 로직 (앱에 통합)**

### 흐름도
```
학생이 프로그램 선택
        ↓
프로그램 위치, 시간 확인
        ↓
"탑승 가능한 버스 있나?" 체크
        ↓
[있음] → 기존 버스 배정
[없음] → 새 버스 노선 자동 생성
        ↓
생성 로직:
  1. 그 프로그램에 등록한 모든 학생의 학교 수집
  2. 학교들을 지역별로 그룹화
  3. 각 지역에서 가장 중심인 학교 선택
  4. 최적 경로 계산 (TSP)
  5. 픽업 시간 설정
  6. 버스 생성 & 학생들에게 알림
        ↓
학생들에게 알림
"당신의 학교에서 픽업 버스가 생겼습니다!"
```

## 📈 **효율성 분석 (KPI)**

### 추적할 지표
```
1. 버스 이용률
   = (예약된 좌석 / 총 좌석) × 100%
   목표: > 80%

2. 평균 탑승 시간
   = (버스에 탄 시간 - 버스 출발 시간)
   목표: < 30분

3. 지역별 접근 공평성
   = 각 구별 버스 이용 학생 수
   목표: 동구·중구 > 50%

4. 프로그램별 버스 활용도
   = (버스 이용 학생 / 총 참여 학생) × 100%
   목표: > 70%
```

### 예시 분석
```
서구 학생:
- 버스 이용률: 30% (자차, 학원 버스 이용)
- 평균 버스 탑승 시간: 25분

동구 학생:
- 버스 이용률: 85% ⭐ (버스 의존도 높음)
- 평균 버스 탑승 시간: 28분

→ 결론: 버스는 동구·중구 학생을 위해 효과적!
```

## 💡 **최적화 팁**

### 1. 같은 지역의 여러 학교 묶기
```
❌ 비효율적:
   13:00 - A중
   13:15 - B중 (같은 동구인데 멀리 떨어짐)

✅ 효율적:
   13:00 - A중
   13:03 - B중 (같은 동구, 가까운 곳)
   → 시간 절약
```

### 2. 우선순위 고려
```
교육격차가 큰 지역(동구, 중구)
→ 버스 탑승 시간 더 짧게
→ 여러 버스 운영

→ 버스 탑승 시간 더 길어도 OK
→ 버스 수 적게
```
